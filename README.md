# AI Usage Monitor

Claude Pro、Gemini Advanced、Google AI Studio の送信回数制限をブラウザ内で記録・表示する Chrome 拡張機能（Manifest V3）。

個人情報・セッショントークンは一切取得・保存しない。すべての処理はローカルで完結する。

---

## 機能

### 送信検知

- claude.ai 上で Claude の API エンドポイント（`/api/.../completion`）への POST を fetch インターセプトで検知し、自動カウントする
- Gemini / AI Studio は引き続きボタンクリック・キーイベントで検知（将来的に fetch 対応予定）

### リセット時刻の自動同期

- Claude の「プラン使用制限」画面を開くと、"X分後にリセット" / "X時間後にリセット" のテキストを DOM から自動検出し、実際のサーバー側リセット時刻を保存する
- 保存されたリセット時刻は、拡張の計算値より優先して表示される

### フローティングウィジェット

- Claude.ai / Gemini / AI Studio を開いている間はバッテリー形式のウィジェットを画面に表示する
- Chrome が開いている間は他のタブでも折りたたみ表示でバッテリー残量を確認できる
- ウィジェットはドラッグで任意の位置に移動でき、位置・折りたたみ状態はページをまたいで保存される
- 折りたたみ表示をダブルクリックで展開／縮小を切り替える
- 展開時は現在のカウント・バッテリーゲージ・充電完了時刻・手動調整ボタン（±1）を表示する

### ポップアップダッシュボード

- 拡張アイコンをクリックすると 3 サービスの状態を一覧表示するダッシュボードが開く
- 各サービスのバッテリーゲージ・カウント・充電完了時刻を表示する
- 送信履歴（タイムスタンプ）を展開して確認・個別削除できる

### 設定

- 各サービスの送信上限数・時間枠（時間）をカスタマイズできる
- 「現在のカウント」を入力してローカルのカウントを実際の状態に合わせられる
- Claude のリセット時刻を手動で同期できる。Claude の使用制限画面に表示される残り分数を入力して保存すると、充電完了時刻が正確な値に更新される。0 を入力すると保存済みのリセット時刻をクリアする

## ファイル構成

```
ai-usage-monitor/
├── manifest.json
├── background.js               # 定期的なログ減衰処理・バッジ更新
├── popup/
│   ├── popup.html
│   ├── popup.css
│   └── popup.js                # ダッシュボード・設定モーダル
└── content-scripts/
    ├── content.js              # ウィジェット表示・DOM リセット時刻検出
    ├── inject.js               # ページコンテキストで fetch/XHR をインターセプト
    └── widget.css
```

## インストール

1. `chrome://extensions` を開く
2. 「デベロッパー モード」をオンにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリックし、このフォルダを選択する

更新後は同画面の更新ボタン（リロードアイコン）を押す。

---

## リセット時刻のズレを修正する方法

1. Claude の画面右上アカウントメニュー →「プラン使用制限」を開く
2. 「現在のセッション」欄に表示されている残り時間を確認する（例: 13分後にリセット）
3. 拡張が自動検出している場合、ウィジェットの「充電完了: HH:MM」が自動で更新される
4. 自動検出されない場合は、拡張アイコン → 設定（歯車アイコン）→ Claude Pro の「リセットまでの残り分数」に残り分数を入力して保存する
